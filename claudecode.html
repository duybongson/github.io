Xây Dựng Skill cho Claude

Giới thiệu

Skill là một tập hợp hướng dẫn — được đóng gói dưới dạng một folder đơn giản — giúp dạy Claude cách xử lý các tác vụ hoặc workflow cụ thể. Skill là một trong những cách mạnh mẽ nhất để tùy chỉnh Claude theo nhu cầu riêng của bạn. Thay vì phải giải thích lại sở thích, quy trình và kiến thức chuyên môn trong mỗi cuộc hội thoại, skill cho phép bạn dạy Claude một lần và hưởng lợi mãi mãi.

Skill đặc biệt mạnh khi bạn có các workflow lặp lại: tạo thiết kế frontend từ spec, nghiên cứu với phương pháp nhất quán, tạo tài liệu theo style guide của team, hoặc điều phối các quy trình nhiều bước.

Hướng dẫn này bao gồm mọi thứ bạn cần biết để xây dựng skill hiệu quả — từ lập kế hoạch và cấu trúc đến kiểm thử và phân phối.

Bạn sẽ học được gì:

  • Yêu cầu kỹ thuật và best practice cho cấu trúc skill
  • Các pattern cho standalone skill và MCP-enhanced workflow
  • Các pattern đã được chứng minh hiệu quả trên nhiều use case
  • Cách kiểm thử, cải tiến và phân phối skill

Đối tượng phù hợp:

  • Developer muốn Claude tuân theo các workflow cụ thể một cách nhất quán
  • Power user muốn Claude tuân theo các workflow cụ thể
  • Team muốn chuẩn hóa cách Claude hoạt động trong tổ chức

Hai lộ trình qua hướng dẫn này

  • Xây dựng standalone skill? Tập trung vào Fundamentals, Planning and Design và Category 1–2.
  • Nâng cấp MCP Integration? Phần "Skills + MCP" và Category 3 dành cho bạn.
Kết quả bạn đạt được: Cuối hướng dẫn, bạn sẽ có thể xây dựng một skill hoàn chỉnh trong một lần ngồi. Dự kiến khoảng 15–30 phút để xây và test skill đầu tiên bằng skill-creator.

Chương 1: Nền tảng cơ bản

Skill là gì?

Một skill là một folder chứa:

  • SKILL.md bắt buộc — Hướng dẫn dạng Markdown với YAML Frontmatter
  • scripts/ tùy chọn — Code thực thi (Python, Bash, v.v.)
  • references/ tùy chọn — Tài liệu tham khảo, được tải khi cần
  • assets/ tùy chọn — Template, font, icon dùng cho output

Nguyên tắc thiết kế cốt lõi

Progressive Disclosure (Tiết lộ dần dần)

Skill sử dụng hệ thống ba tầng:

  • Tầng 1 (YAML Frontmatter): Luôn được tải vào system prompt. Giúp Claude biết khi nào dùng skill mà không tải toàn bộ.
  • Tầng 2 (Phần thân SKILL.md): Được tải khi Claude thấy skill phù hợp. Chứa toàn bộ hướng dẫn chi tiết.
  • Tầng 3 (File liên kết): Các file bổ sung mà Claude có thể khám phá khi cần.

Composability (Khả năng kết hợp)

Claude có thể tải nhiều skill cùng lúc. Skill của bạn nên hoạt động tốt cùng với các skill khác.

Portability (Tính di động)

Skill hoạt động giống nhau trên Claude.ai, Claude Code và API. Tạo skill một lần và nó hoạt động trên mọi nền tảng.

Dành cho MCP Builder: Skills + Connectors

💡 Đang xây standalone skill mà không dùng MCP? Bỏ qua phần này và chuyển đến Chương 2 — bạn luôn có thể quay lại sau.

Nếu bạn đã có MCP Server hoạt động, bạn đã làm xong phần khó nhất. Skill là lớp kiến thức phía trên.

Chương 2: Lập kế hoạch và Thiết kế

Bắt đầu với Use Case

Trước khi viết bất kỳ dòng code nào, hãy xác định 2–3 use case cụ thể mà skill cần hỗ trợ.

Use Case   : Lập kế hoạch Sprint
Trigger    : Người dùng nói "giúp tôi lập kế hoạch sprint"
Các bước   :
  1. Lấy trạng thái dự án hiện tại từ Linear (qua MCP)
  2. Phân tích velocity và capacity của team
  3. Đề xuất ưu tiên task
  4. Tạo task trong Linear với label và estimate phù hợp
Kết quả    : Sprint được lên kế hoạch đầy đủ

Các danh mục Use Case phổ biến

Category 1: Document & Asset Creation

Dùng để tạo output chất lượng cao, nhất quán: tài liệu, bài thuyết trình, ứng dụng, thiết kế, code…

Kỹ thuật chính: Style guide và brand standard được nhúng, cấu trúc template nhất quán, checklist chất lượng, không cần tool bên ngoài.

Category 2: Workflow Automation

Dùng cho quy trình nhiều bước hưởng lợi từ phương pháp nhất quán, bao gồm phối hợp giữa nhiều MCP Server.

Kỹ thuật chính: Workflow từng bước với validation gate, template cho cấu trúc phổ biến, vòng lặp tinh chỉnh.

Category 3: MCP Enhancement

Hướng dẫn workflow để nâng cao quyền truy cập tool mà MCP Server cung cấp.

Kỹ thuật chính: Phối hợp nhiều lệnh MCP theo tuần tự, nhúng chuyên môn lĩnh vực, xử lý lỗi cho vấn đề MCP.

Chương 3: Kiểm thử và Cải tiến

Skill có thể được test ở các mức độ nghiêm ngặt khác nhau:

  • Manual testing trên Claude.ai — Chạy query trực tiếp, lặp nhanh, không cần setup.
  • Scripted testing trên Claude Code — Tự động hóa test case để xác nhận qua các thay đổi.
  • Programmatic testing qua Skills API — Xây evaluation suite chạy có hệ thống.
Pro Tip: Lặp đi lặp lại trên một tác vụ duy nhất trước khi mở rộng. Lặp trên một tác vụ khó cho đến khi Claude thành công, sau đó rút ra thành skill.

1. Triggering Test

Nên trigger:
- "Giúp tôi thiết lập workspace ProjectHub mới"
- "Tôi cần tạo dự án trong ProjectHub"
- "Khởi tạo dự án ProjectHub cho kế hoạch Q4"

KHÔNG nên trigger:
- "Thời tiết ở San Francisco thế nào?"
- "Giúp tôi viết code Python"

Chương 4: Phân phối và Chia sẻ

Mô hình phân phối (Tháng 1/2026)

Người dùng cá nhân nhận skill:

  1. Tải folder skill
  2. Nén folder (nếu cần)
  3. Upload lên Claude.ai qua Settings > Capabilities > Skills
  4. Hoặc đặt vào thư mục skill của Claude Code

Skill cấp tổ chức: Admin có thể triển khai skill toàn workspace. Cập nhật tự động, quản lý tập trung (ra mắt 18/12/2025).

Cách tiếp cận khuyến nghị

## Cài đặt skill [Dịch vụ của bạn]

1. Tải skill:
   - Clone repo: `git clone https://github.com/yourcompany/skills`
   - Hoặc download ZIP từ Releases

2. Cài đặt vào Claude:
   - Mở Claude.ai > Settings > Skills
   - Click "Upload skill"
   - Chọn folder skill (đã nén)

3. Bật skill:
   - Toggle bật skill [Dịch vụ]
   - Đảm bảo MCP Server đã kết nối

4. Test:
   - Hỏi Claude: "Thiết lập dự án mới trong [Dịch vụ]"

Chương 5: Patterns và Xử lý sự cố

Pattern 1: Sequential Workflow Orchestration

## Workflow: Onboard Khách Hàng Mới

### Step 1: Tạo Tài Khoản
Gọi MCP tool: `create_customer`
Tham số: name, email, company

### Step 2: Thiết Lập Thanh Toán
Gọi MCP tool: `setup_payment_method`
Chờ: xác minh phương thức thanh toán

### Step 3: Tạo Subscription
Gọi MCP tool: `create_subscription`
Tham số: plan_id, customer_id (từ Step 1)

### Step 4: Gửi Email Chào Mừng
Gọi MCP tool: `send_email`
Template: welcome_email_template

Pattern 2: Multi-MCP Coordination

### Phase 1: Design Export (Figma MCP)
1. Export tài sản thiết kế từ Figma
2. Tạo specification thiết kế

### Phase 2: Asset Storage (Drive MCP)
1. Tạo folder dự án trong Drive
2. Upload tất cả tài sản

### Phase 3: Task Creation (Linear MCP)
1. Tạo task phát triển
2. Đính kèm link tài sản vào task

### Phase 4: Notification (Slack MCP)
1. Đăng tóm tắt handoff vào #engineering

Skill không upload được

Skill không trigger

Chỉnh sửa field description. Checklist nhanh:

  • Quá chung chung? ("Giúp với dự án" sẽ không hoạt động)
  • Có trigger phrase người dùng thực sự nói không?
  • Có đề cập loại file liên quan không?

Cách debug: Hỏi Claude: "Khi nào bạn sẽ dùng skill [tên skill]?"

Hướng dẫn không được tuân theo

Đảm bảo validate mọi thứ đúng cách.
CRITICAL: Trước khi gọi create_project, xác minh:
- Tên dự án không trống
- Có ít nhất một thành viên team
- Ngày bắt đầu không nằm trong quá khứ

Phụ lục A: Checklist nhanh

Trước khi bắt đầu

  • Xác định 2–3 use case cụ thể
  • Xác định tool cần dùng (tích hợp sẵn hoặc MCP)
  • Đã xem hướng dẫn và skill mẫu
  • Lên kế hoạch cấu trúc folder

Trong quá trình phát triển

  • Folder đặt tên kebab-case
  • File SKILL.md tồn tại (chính xác chính tả)
  • YAML Frontmatter có delimiter ---
  • Field name: kebab-case, không dấu cách, không chữ hoa
  • description bao gồm WHAT và WHEN
  • Không có XML tag (< >)
  • Hướng dẫn rõ ràng và actionable
  • Có xử lý lỗi (error handling)
  • Có ví dụ
  • Reference được link rõ ràng

Trước khi upload

  • Test trigger với tác vụ rõ ràng
  • Test trigger với yêu cầu diễn đạt lại
  • Xác nhận không trigger với chủ đề không liên quan
  • Functional test đạt
  • Tool integration hoạt động (nếu áp dụng)
  • Đã nén thành file .zip

Sau khi upload

  • Test trong cuộc hội thoại thực
  • Giám sát under/over-triggering
  • Thu thập feedback từ người dùng thực

Phụ lục B: YAML Frontmatter

Format đầy đủ

---
name: your-skill-name           # bắt buộc, kebab-case
description: |                  # bắt buộc, < 1024 ký tự
  Mô tả skill làm gì.
  Sử dụng khi người dùng nói "từ khóa" hoặc "cụm từ cụ thể".
license: MIT                    # tùy chọn
compatibility: |                # tùy chọn, 1-500 ký tự
  Yêu cầu MCP Server X phiên bản >= 1.0
metadata:                       # tùy chọn
  author: YourName
  version: 1.0.0
  mcp-server: your-service
---

Quy tắc bảo mật

Cấm trong frontmatter: Dấu ngoặc nhọn XML (< >) và tên skill chứa "claude" hoặc "anthropic" (reserved).

Phụ lục C: Ví dụ Skill hoàn chỉnh

Dưới đây là ví dụ SKILL.md hoàn chỉnh cho skill quản lý dự án:

---
name: project-setup

  "khởi tạo project" hoặc "ProjectHub setup".
license: MIT
metadata:
  author: YourTeam
  version: 1.0.0
  mcp-server: projecthub

# Project Setup Skill

## Instructions

### Step 1: Thu thập thông tin
Hỏi người dùng:
- Tên dự án
- Mô tả ngắn
- Danh sách thành viên team

### Step 2: Tạo workspace
Gọi MCP tool: `create_workspace`
Tham số: name, description

### Step 3: Thêm thành viên
Với mỗi thành viên:
- Gọi: `add_member`
- Tham số: workspace_id, email

### Step 4: Tạo cấu trúc mặc định
Gọi: `create_default_structure`
Bao gồm: boards, labels, milestones

## Examples

### Ví dụ 1: Dự án đơn giản
Người dùng: "Tạo dự án Q4 Marketing cho team 3 người"
Hành động:
  1. Tạo workspace "Q4 Marketing"
  2. Thêm 3 thành viên
  3. Setup cấu trúc mặc định
Kết quả: Workspace sẵn sàng với link chia sẻ

## Troubleshooting

Error: "Member not found"
Nguyên nhân: Email không tồn tại trong hệ thống
Giải pháp: Xác minh email, mời thành viên qua Settings > Invite
